# Product

## Register

product

## Naming

**PulseRadar** — 脉冲雷达。

「Pulse」既是市场的脉搏（盯住它的心跳），也是信号脉冲（异动就是一次电脉冲）。与 FinanceRadar（盘前作战地图）形成姊妹产品：一个是出发前看地图，一个是行驶中的雷达预警。

## Users

A 股个人投资者，尤其是日内交易和短线操作者。核心场景：**交易时间（09:30-15:00）的盘中实时监控**。

痛点不是「看不到行情」——同花顺/东方财富已经有实时行情了——而是「5000 多只股票，我不可能同时盯住所有标的；等我发现某只票在拉升的时候，往往已经错过了最佳介入点。即使设了自选股提醒，也只能盯住已经关注的票，而真正的机会往往来自预期之外的地方。」

这个「从 5000+ 只票中实时筛选出值得关注的异动信号」的苦力活，就是我们要替代的。

## Product Purpose

PulseRadar 是一个 **AI 盘中异动雷达**（不是行情软件，也不是量化交易系统）。

它要回答的核心问题只有一个：**「现在有什么值得我注意的异动？」**

为了回答这个问题，它帮用户完成以下工作：

1. **全市场扫描** → 每 5 秒扫一遍全部 A 股，不遗漏任何一个角落
2. **多维信号融合** → 不是单纯看涨幅，而是涨速+量比+板块共振+技术形态的综合判断
3. **过滤噪音** → 通过量化评分系统过滤庄股、ST、低流动性等假信号
4. **主动推送** → 不需要你一直盯着屏幕，发现异动直接弹桌面通知

成功标准：用户开盘时说一句「开始盯盘」，然后可以去做别的事，有异动时桌面弹通知扫一眼就够了。收盘后可以回顾今天所有异动，复盘哪些抓住了、哪些错过了。

## Relationship with FinanceRadar

两个产品是「盘前+盘中」的完整作战链条：

- **FinanceRadar**（盘前作战地图）→ 回答「今天我该怎么办？」→ 每天 08:30 生成静态报告
- **PulseRadar**（盘中异动雷达）→ 回答「现在有什么值得注意的？」→ 交易时间实时运行

FinanceRadar 告诉你「今天存储链可能高开，关注兆易创新」，PulseRadar 在盘中发现「兆易创新 10:15 放量突破 VWAP，涨速加快」时推送通知——两者串联起来就是完整的发现→确认→行动链路。

## Brand Personality

敏锐、克制、可信赖 — 像一个经验丰富的操盘手坐在你旁边，平时安静不打扰，但一有异动立刻拍你肩膀告诉你「这个值得看一眼」。不是那种什么都喊的股票群，也不是冷冰冰的程序化交易终端。每一次提醒都经过多维验证，宁可漏报也不误报。

## Anti-references

1. **股票群/直播间** — 「快快快，冲！」不经分析的情绪化喊单，信噪比极低
2. **行情软件的涨幅排行** — 简单的按涨幅排序，没有信号过滤和多维度融合，噪音太多
3. **量化交易系统** — 全自动下单，对个人投资者来说是黑盒，不透明、不可控
4. **传统条件选股** — 只能设静态条件（如涨幅>5%），无法捕捉「正在发生」的动态异动过程

## Design Principles

1. **宁缺毋滥，每一次推送都值得你放下手里的事** — 通过多维评分+置信度过滤，确保推送的信号质量。用户收到 10 条通知，至少 7 条是有意义的。
2. **信号可解释** — 每个异动推送不只告诉你「涨了」，而是告诉你「为什么这次涨和以前不一样」：放量+VWAP突破+板块共振，用户自己判断要不要行动。
3. **控制器而非自动驾驶** — 我们提供雷达和预警，但决策权始终在用户手中。不自动交易，不催促行动，只呈现事实和分析。
4. **状态可感知** — 用户随时知道雷达是否在运行、扫描了多少只票、发现了多少信号、过滤了多少噪音。不是一个看不见摸不着的后台进程。

## Core Architecture

### 技术栈

- **数据源**：AKShare（Python 库），底层对接东方财富数据接口
  - `stock_zh_a_spot_em` — 全市场实时行情（5000+ 只，含涨速/量比/换手率等 23 个字段）
  - `stock_zt_pool_em` — 涨停股池（封单量/炸板次数/连板数）
  - `stock_zt_pool_zbgc_em` — 炸板股池
  - `stock_changes_em` — 东方财富盘口异动信号
  - `stock_board_industry_spot_em` — 行业板块实时行情
  - `stock_board_concept_spot_em` — 概念板块实时行情
- **运行时**：Python 后台进程（由 Skill 控制启停）
- **通知**：`catdesk notify` 系统桌面通知
- **数据存储**：本地 JSON 日志（每日异动记录）

### 轮询策略

- 核心行情数据：**每 5 秒**轮询一次（接近 Level 1 行情的 3 秒刷新频率）
- 涨停池/板块数据：**每 30 秒**轮询一次（变化频率较低，节省请求量）
- 技术指标（RSI/布林带等）：**每 60 秒**计算一次（基于分钟级 K 线）
- 交易时间外自动暂停：仅 09:25-15:05 运行

## Signal System（信号体系）

### P0 · 核心信号（必须实现，直接触发告警的基础条件）

**1. 涨速异动**
- 数据来源：`stock_zh_a_spot_em` 的「涨速」和「5分钟涨跌」字段
- 触发条件：涨速 > 阈值（可配置，默认 2%/min），或 5 分钟涨幅 > 3%
- 同时需满足量比 > 1.5（排除缩量异动）

**2. 量比突变**
- 数据来源：`stock_zh_a_spot_em` 的「量比」字段
- 触发条件：量比 > 3（显著放量）或量比 > 8（极端放量）
- 用于独立检测「突然放量但涨幅尚不明显」的早期信号

**3. 涨停板/炸板监控**
- 数据来源：`stock_zt_pool_em`、`stock_zt_pool_zbgc_em`
- 触发事件：新增涨停（首次封板）、涨停打开（炸板）、封单量大幅变化
- 输出：封单金额、炸板次数、连板数、首封时间

**4. 基础过滤器**
- 排除 ST/*ST 股票（名称前缀匹配）
- 排除流通市值 < 20 亿元
- 排除日成交额 < 5000 万元
- 排除换手率 < 0.3%
- 排除已涨停/跌停（涨跌幅 > 9.8% 或 < -9.8%，创业板/科创板 > 19.8%）

### P1 · 增强信号（核心增值，提升信号可信度和解释力）

**5. VWAP 突破**
- 计算方式：累计成交额 ÷ (累计成交量 × 100)
- 触发条件：股价从 VWAP 下方运行 > 30 分钟后放量上穿 VWAP
- 价值：机构交易员最常用的日内基准，突破 VWAP 说明价格趋势在转变

**6. 板块共振**
- 数据来源：`stock_board_industry_spot_em`、`stock_board_concept_spot_em`
- 逻辑：当个股触发异动时，检查其所在板块是否同步走强
- 共振加分：个股异动 + 板块涨幅 top 10 → 置信度 +15%
- 价值：板块共振的个股拉升，持续性远好于独立拉升

**7. 连板梯队追踪**
- 数据来源：`stock_zt_pool_em` 的「连板数」字段
- 输出：当前市场最高连板数、各层连板股数量
- 价值：连板梯队高度反映市场投机情绪温度

**8. 封单强度分析**
- 计算方式：封单金额 ÷ 流通市值
- 分级：> 5% 强封、2%-5% 中等、< 2% 弱封
- 价值：预判涨停板次日能否延续

### P2 · 精细化信号（技术指标，进一步降低误报率）

**9. 布林带突破**
- 基于 5 分钟 K 线的 20 周期布林带
- 触发：价格突破上轨 + 放量确认
- 预热期：开盘后 100 分钟（约 10:50 后信号可用）

**10. 瞬时大单推断**
- 计算方式：相邻两次轮询的成交额差值
- 触发条件：5 秒内成交额增量 > 500 万元（可配置）
- 价值：间接检测大单扫货行为

**11. 尾盘异动标记**
- 14:30 后触发的异动信号自动打上「尾盘」标签
- 尾盘拉升降低置信度（-20%），因为可能是操纵行为
- 但仍然推送，交由用户判断

**12. RSI 极端值预警**
- 基于 5 分钟 K 线的 14 周期 RSI
- RSI > 85 标记「短期超买」，RSI < 15 标记「短期超卖」
- 不作为独立信号，仅作为标签附加在其他信号上

## Scoring System（综合评分系统）

每个触发异动的股票都会计算一个综合异动分数（0-100），只有超过阈值（默认 60 分）才推送通知。

### 信号权重

| 信号类型 | 权重 | 说明 |
|---------|------|------|
| 涨速异动 | 20% | 核心触发条件 |
| 成交量突变 | 15% | 放量是拉升可信度的基础 |
| VWAP 突破 | 15% | 机构级基准信号 |
| 板块共振 | 10% | 板块联动增强持续性判断 |
| 封单/涨停 | 8% | 打板策略核心指标 |
| 布林带突破 | 7% | 统计异常确认 |
| 大单推断 | 5% | 资金面辅助确认 |
| RSI 状态 | 5% | 超买超卖参考 |
| MACD 方向 | 5% | 趋势动量参考 |
| 尾盘时段调整 | 5% | 时间维度修正 |
| 连板/情绪 | 5% | 市场情绪参考 |

### 置信度调整因子

**正向因子（提升置信度）：**
- 流通市值 > 50 亿元：+10%
- 板块共振确认：+15%
- 多个独立信号同时触发：每增加一个 +10%
- 量比 > 5（强放量）：+10%

**负向因子（降低置信度）：**
- 流通市值 < 30 亿元：-10%
- 近期频繁异动但不持续（「狼来了」）：-15%
- 尾盘时段（14:30 后）：-20%
- 庄股特征（换手率极低 + 筹码集中）：-30%

### 最终计算

```
raw_score = Σ(signal_i × weight_i × signal_strength_i)
confidence = base_confidence(0.5) × Π(adjustment_factors)
final_score = raw_score × confidence × 200  // 归一化到 0-100
```

## User Interaction（用户交互）

### Skill 触发词

- "开始盯盘" / "启动雷达" / "PulseRadar 启动" → 启动后台监控
- "停止盯盘" / "关闭雷达" / "PulseRadar 停止" → 停止后台监控
- "雷达状态" / "盯盘状态" → 查看运行状态（运行时长、扫描次数、推送数量）
- "今日异动" / "异动汇总" → 查看今天截至目前所有触发过的异动记录
- "调整灵敏度" → 调整推送阈值（更敏感 = 更多推送 = 更多噪音；更迟钝 = 更少推送 = 可能漏掉机会）
- "盯住 XXX" / "关注 XXX" → 将股票加入自选盯梢列表
- "取消盯住 XXX" → 将股票从自选盯梢列表移除
- "我的自选" / "盯梢列表" → 查看当前自选股列表及其实时状态

### 通知格式

桌面通知需要在极短的文字内传达关键信息：

```
标题：🔴 快速拉升 · 兆易创新 (603986)
正文：涨幅 +5.2% | 量比 4.3 | 放量突破VWAP | 存储芯片板块共振
      异动评分 78/100
```

```
标题：⚠️ 涨停炸板 · XX科技 (300xxx)
正文：炸板 2 次 | 封单仅 1.2% | 3连板 → 谨慎追高
      异动评分 65/100
```

### 双轨模式：全市场扫描 + 自选股盯梢

PulseRadar 同时运行两套检测逻辑：

**全市场扫描（Market Scan）**
- 扫描范围：全部 A 股（5000+ 只）
- 推送阈值：高（默认 60 分），宁缺毋滥
- 目的：发现预期之外的机会，捕捉你没关注过的票

**自选股盯梢（Watchlist Guard）**
- 扫描范围：用户配置的自选股列表（通常 10-30 只）
- 推送阈值：低（默认 30 分），轻微异动也通知
- 额外能力（全市场扫描做不到的）：
  - **关键价位提醒**：价格触及用户设定的目标价/支撑位/压力位
  - **止损预警**：跌破用户设定的止损线时立即通知
  - **日内走势摘要**：自选股即使没有异动，午盘休市时也推送一份状态小结
- 通知标记：自选股的通知会标注 ⭐ 以区分全市场扫描的通知

自选股列表支持动态增删，用户可以随时说「盯住兆易创新」或「取消盯住XX」来调整。

### 日志与复盘

每日异动记录自动保存为 JSON 日志：
- 路径：`<output_dir>/pulse-radar-{YYYY-MM-DD}.json`
- 内容：时间戳、股票代码/名称、触发信号、评分、当时价格
- 收盘后可触发「今日异动复盘」，回顾哪些信号后续走势验证了判断

## Configuration（配置项）

```json
{
  "poll_interval": 5,
  "push_threshold": 60,
  "sensitivity": "medium",
  "filters": {
    "exclude_st": true,
    "min_market_cap": 2000000000,
    "min_daily_turnover": 50000000,
    "min_turnover_rate": 0.3
  },
  "watchlist": {
    "enabled": true,
    "stocks": [
      { "code": "603986", "name": "兆易创新", "stop_loss": null, "target_price": null },
      { "code": "002049", "name": "紫光国微", "stop_loss": 58.00, "target_price": 72.00 }
    ],
    "push_threshold": 30,
    "midday_summary": true
  },
  "notifications": {
    "desktop": true,
    "daxiang": false,
    "email_summary": false,
    "sound": true
  },
  "output_dir": "~/Desktop/PulseRadar"
}
```

- `poll_interval`：轮询间隔（秒），默认 5，最小 3
- `push_threshold`：全市场扫描推送阈值（0-100），默认 60，越高越严格
- `sensitivity`：灵敏度预设（low / medium / high），会联动调整各项阈值
- `filters`：基础过滤器配置（仅作用于全市场扫描，自选股不受过滤器限制）
- `watchlist`：自选股盯梢配置
  - `enabled`：是否启用自选股盯梢
  - `stocks`：自选股列表，每只可选设 `stop_loss`（止损价）和 `target_price`（目标价）
  - `push_threshold`：自选股推送阈值，默认 30（比全市场更敏感）
  - `midday_summary`：午盘休市时是否推送自选股状态小结
- `notifications`：通知方式配置
  - `desktop`：macOS 桌面通知（实时异动提醒，默认开启）
  - `daxiang`：大象消息推送（离开电脑时的移动端备用，默认关闭）
  - `email_summary`：邮件汇总（午盘/收盘时发送异动汇总邮件，默认关闭）
  - `sound`：通知声音

## MVP Scope（最小可行版本）

**MVP 只实现 P0 信号 + 桌面通知 + 基础日志**，目标是先跑通「启动→扫描→检测→推送」的完整链路：

1. Python 后台脚本：5 秒轮询 AKShare 全市场数据
2. P0 信号检测：涨速异动 + 量比突变 + 基础过滤器
3. 桌面通知：`catdesk notify` 推送
4. Skill 控制：启动/停止/查看状态
5. JSON 日志：记录每次异动

MVP 验证通过后，再逐步加入 P1（VWAP/板块共振/涨停监控）和 P2（技术指标）。

**MVP v0.2（自选股盯梢）：**

6. 自选股配置与低阈值异动检测
7. 关键价位/止损提醒
8. 午盘自选股状态小结

## Accessibility & Compliance

- 通知文本包含完整信息（不仅靠颜色区分涨跌，也用 +/- 符号）
- 免责声明：所有推送均标注「AI 信号仅供参考，不构成投资建议」
- 数据来源：公开免费接口，个人研究用途，控制请求频率（≥ 5 秒间隔）
- 不自动执行任何交易操作
